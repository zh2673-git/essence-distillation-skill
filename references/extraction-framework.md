# 三阶蒸馏方法论

> 从原始信息到可运行的认知操作系统，核心方法论

---

## 一、心智模型识别的三重验证

一个论点要被认定为「心智模型」而非「随口一说」，必须通过三重验证：

### 验证1: 跨域复现

同一个思维框架出现在此人讨论的至少2个不同领域。

例：纳瓦尔的「杠杆」概念——
- 在财富创造中谈杠杆（代码、媒体、资本、劳动力）
- 在个人成长中谈杠杆（特定知识 + 杠杆 = 复利）
- 在职业选择中谈杠杆（选择有杠杆效应的工作）
→ 跨3个领域复现 → 这是真正的心智模型

### 验证2: 有生成力

用这个模型可以推断此人对新问题的可能立场。

例：如果芒格的「逆向思维」是心智模型——
- 面对「如何成功」→ 他会先想「如何确保失败」
- 面对「如何投资」→ 他会先想「如何亏光所有钱」
→ 能生成新推断 → 这是真正的心智模型

### 验证3: 有排他性

不是所有聪明人都会这样想，这个模型体现了此人的独特视角。

例：「反脆弱」是塔勒布的，不是所有人都这样看世界
→ 有区分度 → 这是值得提炼的心智模型

**降级规则**：
- 只通过1重验证 → 降级为「决策启发式」
- 0重验证都没过 → 不纳入

## 二、根因追溯法（本质蒸馏独有）

### 2.1 因果链构建

每个心智模型必须追溯其形成根因，构建「经历→认知→模型」的因果链：

```
关键经历/事件
    ↓ 形成认知
认知模式（此人开始这样看世界）
    ↓ 反复验证固化
心智模型（稳定的思维框架）
```

### 2.2 根因的五个维度

| 维度 | 追问 | 示例（芒格） |
|------|------|------------|
| **教育训练** | 什么学科塑造了他的思维？ | 律师训练→逆向思考；气象学→多元因素分析 |
| **关键人物** | 谁对他影响最大？ | 富兰克林→实用主义；格雷厄姆→价值投资 |
| **重大挫折** | 什么失败让他改变了看法？ | 早期投资失败→能力圈概念 |
| **文化土壤** | 什么环境孕育了这种思维？ | 美国中西部务实文化→反浮夸 |
| **演化转折** | 什么时候他改变了核心观点？ | 从纯投资→多元思维（受心理学影响） |

### 2.3 根因的可信度分级

| 级别 | 标准 | 标注方式 |
|------|------|---------|
| **确定根因** | 此人亲口说过「因为X经历，我形成了Y想法」 | 🟢 确定 |
| **强推测** | 经历和模型高度吻合，有多个间接证据 | 🟡 强推测 |
| **弱推测** | 逻辑上说得通但缺乏直接证据 | 🟠 弱推测 |
| **无法追溯** | 找不到合理的形成原因 | ⚪ 未知 |

## 三、场景路由设计（本质蒸馏独有）

### 3.1 路由表构建

为每个蒸馏对象构建场景路由表，定义不同问题类型应该调用哪些心智模型组合：

```
路由表：
├── 场景A（如：投资决策）→ [模型1] + [模型2] + [启发式3]
├── 场景B（如：人生选择）→ [模型4] + [启发式1]
├── 场景C（如：创造性工作）→ [模型5] + [模型1]
└── 场景D（如：风险判断）→ [模型2] + [模型3] + [启发式5]
```

### 3.2 路由设计原则

1. **不重叠**：每个场景的主模型组合应该有差异性
2. **有依据**：路由分配必须基于此人在该场景的实际表现
3. **有默认**：无法匹配场景时，使用此人的核心模型组合
4. **有边界**：明确标注哪些场景此人不擅长

### 3.3 路由验证

用3个已知场景验证路由表的准确性：
- 此人在投资场景用了哪些模型？→ 对比路由表的投资决策路由
- 此人在人生选择场景用了哪些模型？→ 对比路由表的人生选择路由
- 不匹配则调整路由表

## 四、表达DNA的量化方法

### 4.1 句式指纹

从此人的长文/演讲中随机抽取20个段落，统计：

| 维度 | 测量方式 |
|------|---------|
| 平均句长 | 字数/句数 |
| 疑问句比例 | 疑问句数/总句数 |
| 类比密度 | 类比数/千字 |
| 第一人称使用率 | 「我」出现频率 |
| 确定性语气比例 | 「一定」「显然」vs「也许」「可能」 |
| 转折频率 | 「但是」「然而」「不过」/千字 |

### 4.2 风格标签

从以下维度打标：

```
正式 ←→ 口语
抽象 ←→ 具体
谨慎 ←→ 断言
学术 ←→ 通俗
长句 ←→ 短句
铺垫型 ←→ 结论先行
数据驱动 ←→ 叙事驱动
```

### 4.3 禁忌词和口癖

- 此人从不用的词 → 生成Skill时也不用
- 此人的口癖/高频表达 → 适度使用（太多变成模仿秀）

## 五、坡度理解在Skill生成中的应用

### 5.1 Skill结构必须遵循坡度

生成的Skill必须按坡度理解组织内容：

```
坡度1（30秒抓住核心）：
├── 一句话定义此人的认知本质
├── 身份卡（50字第一人称介绍）
└── 核心心智模型列表（名称+一句话）

坡度2（2分钟理解框架）：
├── 心智模型之间的关系图
├── 场景路由表
└── 根因追溯摘要

坡度3（按需深入）：
├── 每个心智模型的完整证据链
├── 每个决策启发式的案例
├── 表达DNA的详细规则
└── 诚实边界和局限
```

### 5.2 每次回答也遵循坡度

```
用户提问 → Skill回应：
├── 先用核心心智模型「点出概念」（1-2句话定位问题本质）
├── 再用因果链「建立联系」（展示问题与此人思维框架的关联）
└── 最后用决策启发式「详细解释」（给出具体的判断和建议）
```

## 六、四层基础设施校验

每个生成的Skill必须通过四层校验：

| 层面 | 校验内容 | 通过标准 |
|-----|---------|---------|
| **数据规矩** | 心智模型定义是否清晰？边界是否明确？ | 每个模型有一句话定义+局限说明 |
| **数据存储** | 调研来源是否充分？一手占比>60%？ | 至少3个一手来源 |
| **数据流转** | 从问题到判断的推理链是否完整？ | 路由表覆盖≥3个场景 |
| **接口层** | 表达DNA是否可执行？触发条件是否明确？ | 读一段输出能识别是谁 |

## 七、矛盾处理原则

矛盾是人格的核心特征，不是需要修复的Bug。

### 三种矛盾类型

1. **时间性矛盾**（观点演化）
   - 处理：记录演化轨迹，标注「早期」「近期」，Skill中以近期为主但提及演化

2. **领域性矛盾**（不同场景不同规则）
   - 处理：分领域记录，不强求统一

3. **本质性张力**（价值观内在冲突）
   - 处理：明确记录为「核心张力」

### 错误的处理方式

- ❌ 选一边忽略另一边
- ❌ 编一个调和的解释
- ❌ 假装矛盾不存在

---

## 八、信息不足时的处理

| 情况 | 处理 |
|------|------|
| 某个维度公开信息很少 | 在Skill中标注「信息不足，此维度为推测」 |
| 只有二手信息 | 降低置信度，标注为「据[来源]报道」 |
| 信息相互矛盾且无法判断 | 并列呈现，让用户自行判断 |
| 此人刻意不公开某方面 | 尊重边界，注明「此人对这个话题保持沉默」 |
| 根因无法追溯 | 标注为⚪未知，不编造因果链 |

## 九、质量自检清单

### 心智模型
- [ ] 每个模型都有至少2个不同领域的证据？
- [ ] 模型数量在3-7个之间？
- [ ] 每个模型都有明确的应用场景和局限？
- [ ] 模型之间有张力但不矛盾？

### 根因追溯（本质蒸馏独有）
- [ ] 每个核心模型都有形成根因？
- [ ] 根因的可信度已分级标注？
- [ ] 至少50%的模型有🟢确定或🟡强推测根因？

### 场景路由（本质蒸馏独有）
- [ ] 路由表覆盖≥3个场景？
- [ ] 每个场景的模型组合有依据？
- [ ] 有默认路由和边界说明？

### 表达DNA
- [ ] 读起来有辨识度，不像通用AI？
- [ ] 没有过度模仿变成caricature？
- [ ] 抓住了核心特征而非表面模仿？

### 坡度完整性（本质蒸馏独有）
- [ ] 坡度1：30秒能抓住此人认知本质？
- [ ] 坡度2：2分钟能理解心智模型之间的关系？
- [ ] 坡度3：深入任何模型都有完整证据链？

### 诚实边界
- [ ] 明确写了做不到什么？
- [ ] 标注了信息源和调研时间？
- [ ] 承认了信息不足的维度？

### 整体
- [ ] 用此人的眼睛看一个新问题，能得到有价值的视角？
- [ ] 不是此人原话的拼凑，而是框架的运行？
- [ ] 删掉名字后，还能认出这是谁的思维方式？
