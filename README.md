# 本质蒸馏 | Essence Distillation

> 三阶认知提炼系统：输入人名，输出可运行的认知操作系统

**是什么 → 为什么 → 怎么做**

---

## 这是什么

本质蒸馏是一个认知提炼系统。它不只是提取「他怎么想」，而是追溯「他为什么这么想」，最终构建「怎么运行他的认知系统」。

提取的不只是认知的**快照**，而是认知的**操作系统**——包含源码（为什么）、运行时（怎么想）、API（怎么用）。

### 与表层蒸馏的区别

| 层次 | 提取内容 | 举例 |
|------|---------|------|
| 表层 | HOW they think | 「倪海厦用经方治病」 |
| 本质 | WHY → HOW → HOW TO RUN | 「易经启蒙→阴阳归因→六经辨证→经方选药」 |

---

## 核心方法论

**蒸馏公式**：三阶方法论 × 坡度理解 × 类-属性-方法-路由

### 三阶蒸馏

```
第一阶：是什么 → 认知本质（一句话定义 + 核心心智模型）
第二阶：为什么 → 根因追溯（经历→认知→模型的因果链）
第三阶：怎么做 → 系统建模（类-属性-方法-路由的完整操作系统）
```

### 坡度理解

所有输出遵循三层渐进结构：

- **坡度1**（30秒）：一句话抓住认知本质
- **坡度2**（2分钟）：心智模型关系 + 场景路由 + 根因摘要
- **坡度3**（按需深入）：完整证据链 + 递归子模型

### 类-属性-方法-路由

| 概念 | 对应 | 作用 |
|------|------|------|
| Class | 认知系统 | 整体定义 |
| 属性 | 心智模型 | 他怎么看世界 |
| 方法 | 决策启发式 | 他怎么判断 |
| 路由 | 场景适配 | 不同场景用什么策略 |
| 接口 | 表达DNA | 他怎么表达 |

## 执行流程

```
Phase 0  入口分流
  ├── 明确人名 → 直接路径（需求澄清）
  └── 模糊需求 → 诊断路径（从需求反推蒸馏对象）

Phase 0.5  创建Skill目录

Phase 1   多源信息采集（7个Agent并行）
  ├── Agent 1: 著作与系统思考
  ├── Agent 2: 长对话与即兴思考
  ├── Agent 3: 碎片表达与风格DNA
  ├── Agent 4: 他者视角与批评
  ├── Agent 5: 决策记录与行动
  ├── Agent 6: 人物时间线
  └── Agent 7: 根因分析 ← 本质蒸馏独有

Phase 1.5 调研Review检查点

Phase 2   三阶蒸馏 ← 核心创新
  ├── 2.1 是什么：认知本质
  ├── 2.2 为什么：根因追溯
  └── 2.3 怎么做：系统建模

Phase 3   构建Skill

Phase 4   质量验证（三重验证 + 四层校验 + 实战测试）

Phase 5   递归深入（核心模型可无限递归三阶蒸馏）
```

## 蒸馏实例

> 所有实例存放在 `examples/` 目录下，每个子目录包含完整的Skill（SKILL.md + references + few-shot）。

### 倪海厦

**一句话本质**：以阴阳五行为底层操作系统、以经方为执行程序、以"站在外面看到里面"为诊断哲学的归纳式医学认知系统

**5个核心心智模型**：

| 模型 | 一句话 | 根因 |
|------|--------|------|
| 阴阳归因 | 一切疾病皆可归因为阴阳失衡 | 车祸卧床读易经→由道入医 |
| 经方至上 | 伤寒论六经辨证是中医核心 | 师承经方→临床反复验证 |
| 治人不治病 | 中医治的是人的状态而非病名 | 易经整体观→非科班不受病名训练 |
| 归纳优于演绎 | 中国文化强在归纳万变不离其宗 | 唐容川影响→西医越分越细病人越多 |
| 术数入医 | 易经/命理/风水与中医底层逻辑相通 | 算命起家→五术兼修→文化闭环 |

详见 [examples/nihaixia.skill/](examples/nihaixia.skill/)

**5个核心心智模型**：

| 模型 | 一句话 | 根因 |
|------|--------|------|
| 自知无知 | 真正的智慧是认识到自己知识的边界 | 德尔斐神谕验证→一生质问反复确认 |
| 诘问法 | 通过追问揭示信念矛盾 | 与智者派交锋→揭示矛盾比提供答案更有效 |
| 德性即知识 | 所有恶行源于对善的无知 | 观察到作恶者以为在求善 |
| 灵魂优先 | 照顾灵魂比照顾身体/财富更重要 | 从自然哲学转向→用死亡验证 |
| 宁受不义不行不义 | 遭受不义是恶，但行不义是更大的恶 | 拒绝三十僭主命令→用死亡验证 |

## 如何使用

### 在 Trae IDE 中

1. 将 `SKILL.md` 放入项目的 `.claude/skills/` 或 `.trae-cn/skills/` 目录
2. 对话中提到「蒸馏」「提炼XX的思维」即可触发
3. 蒸馏结果会自动生成到 `examples/` 目录

### 独立使用

1. 阅读根 `SKILL.md` 了解完整流程
2. 参考 `references/extraction-framework.md` 理解方法论细节
3. 参考 `references/skill-template.md` 了解输出格式
4. 以 `examples/nihaixia.skill/` 为范例，自行蒸馏

## 本质蒸馏的独有能力

| 能力 | 说明 |
|------|------|
| **根因追溯** | 每个心智模型追溯其形成经历和认知固化点，不只「他怎么想」还追溯「他为什么这么想」 |
| **场景路由** | 不同问题类型调用不同模型组合，可执行的路由机制 |
| **递归深入** | 核心模型可无限递归三阶蒸馏，直到模型足够基础无法再分解 |
| **坡度理解** | 所有输出遵循三层渐进结构，30秒→2分钟→按需深入 |
| **矛盾保留** | 矛盾是人格的核心特征，不是需要修复的Bug |
